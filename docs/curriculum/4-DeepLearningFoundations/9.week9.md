### Week 9 — Day-by-Day Breakdown

## Week 9 Theme: Deep Learning Foundations — Neural Networks Basics 🧠

**Goal (by end of Week 9):**

- Understand what a **neural network** is in practical terms: layers, neurons, weights, biases, activations
- Connect neural networks to what we already know from ML: **linear models + non-linearity + optimization**

## Week 9 — Day 1 🔜  

- ***Focus: Why Neural Networks Exist + From Linear Models to Learned Representations***
- Revisit the limits of plain linear models:
  - a linear model learns one weighted sum and one direct mapping to the output
  - this is often too rigid for more complex patterns
- Build the motivation for neural networks:
  - stacked transformations
  - hidden layers that learn intermediate representations
  - ability to model non-linear relationships
- Establish the first important truth:
  - stacking only linear layers still gives a linear transformation
  - non-linearity is what gives the network real expressive power
- Frame the mental model correctly:
  - a neural network is not magic
  - it is a parameterized function trained by optimization

✅ **Outcome:** We understand *why* neural networks are needed before touching implementation.

**Checkpoint Questions:**

- What limitation of a linear model motivates neural networks?
- Why do stacked linear layers still collapse into one linear transformation?


## Week 9 — Day 2 🔜  

- ***Focus: Anatomy of a Neural Network (Layers, Neurons, Weights, Biases)***
- Define the core building blocks:
  - input layer
  - hidden layer(s)
  - output layer
  - neuron
  - weights
  - bias
- Map one neuron to familiar math:
  - weighted sum
  - add bias
  - apply activation
- Connect scalar intuition to matrix form:
  - scalar view helps with meaning
  - matrix view explains efficient implementation
- Clarify terminology carefully:
  - neurons are simple computation units, not mystical little gremlins
  - a layer is a collection of those units operating together

✅ **Outcome:** We can describe what each component of a neural network does without hand-waving.

**Checkpoint Questions:**

- What does a single neuron compute?
- Why do we need both weights and bias?


## Week 9 — Day 3 🔜  

- ***Focus: Forward Propagation + Activation Functions***
- Trace how predictions are produced:
  - inputs enter the network
  - each layer transforms them
  - final layer produces logits, probabilities, or numeric predictions depending on the task
- Study forward propagation as a repeated pattern:
  - matrix multiply
  - add bias
  - apply activation
- Introduce common activation functions:
  - **ReLU**
  - **sigmoid**
  - **tanh**
- Explain why activations matter:
  - without non-linearity, depth adds no real expressive power
  - activation functions let the network learn richer mappings
- Clarify practical intuition:
  - ReLU is common in hidden layers
  - sigmoid often appears in binary output settings
  - tanh is useful for intuition and appears in some architectures

✅ **Outcome:** We understand how a network turns input into output and why non-linearity is essential.

**Checkpoint Questions:**

- What happens during forward propagation?
- Why is ReLU often preferred over sigmoid in hidden layers?


## Week 9 — Day 4 🔜  

- ***Focus: Loss Functions + Gradient Descent in Neural Networks***
- Define **loss** as the signal that tells the model how wrong it is during training
- Connect task type to typical loss:
  - **binary cross-entropy** for binary classification
  - **mean squared error** for regression
- Distinguish clearly:
  - **loss** is used for optimization
  - **metrics** are used for evaluation and reporting
- Revisit gradient descent in this new setting:
  - many parameters instead of just a few
  - the same core idea still applies: adjust parameters to reduce loss
- Build intuition for the optimization landscape:
  - the model moves through parameter space trying to reduce error

✅ **Outcome:** We understand what the network is trying to minimize and how gradient-based learning fits into deep learning.

**Checkpoint Questions:**

- Why do we separate loss from metrics?
- What is gradient descent trying to achieve during training?


## Week 9 — Day 5 🔜  

- ***Focus: Backpropagation + Chain Rule Intuition***
- Introduce backpropagation as the mechanism that computes how each parameter contributed to error
- Keep the explanation intuition-first:
  - make prediction
  - compute loss
  - send error information backward through the network
  - compute gradients
  - update parameters
- Connect backpropagation to the **chain rule**:
  - each layer depends on the previous one
  - gradients must be propagated through those dependencies
- Explain why this matters:
  - without backprop, training a multi-layer network would be wildly impractical
- Keep the scope sane:
  - we do not need full symbolic derivations for giant networks yet
  - we need clear mechanics first

✅ **Outcome:** We understand what backpropagation does and why the chain rule sits at the heart of deep learning.

**Checkpoint Questions:**

- What does backpropagation compute?
- Why is the chain rule essential in multi-layer models?

### Examples

#### Example 1 — A Tiny Feedforward Network

```python
import torch
from torch import nn

class TinyNet(nn.Module):
    def __init__(self, input_dim: int):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(input_dim, 16),
            nn.ReLU(),
            nn.Linear(16, 8),
            nn.ReLU(),
            nn.Linear(8, 1),
        )

    def forward(self, x):
        return self.net(x)
```

#### Example 2 — One Training Step

```python
def train_one_epoch(model, dataloader, loss_fn, optimizer, device):
    model.train()
    running_loss = 0.0

    for X_batch, y_batch in dataloader:
        X_batch = X_batch.to(device)
        y_batch = y_batch.to(device)

        optimizer.zero_grad()
        logits = model(X_batch)
        loss = loss_fn(logits, y_batch)
        loss.backward()
        optimizer.step()

        running_loss += loss.item()

    return running_loss / len(dataloader)

```

#### Example 3 — Evaluation Pass

```python
@torch.no_grad()
def evaluate(model, dataloader, loss_fn, device):
    model.eval()
    running_loss = 0.0

    for X_batch, y_batch in dataloader:
        X_batch = X_batch.to(device)
        y_batch = y_batch.to(device)

        logits = model(X_batch)
        loss = loss_fn(logits, y_batch)
        running_loss += loss.item()

    return running_loss / len(dataloader)
```

#### Example 4 — Minimal Training Loop Skeleton

```python
num_epochs = 20

for epoch in range(num_epochs):
    train_loss = train_one_epoch(model, train_loader, loss_fn, optimizer, device)
    val_loss = evaluate(model, val_loader, loss_fn, device)
    print(f"epoch={epoch+1:02d} train_loss={train_loss:.4f} val_loss={val_loss:.4f}")
```
