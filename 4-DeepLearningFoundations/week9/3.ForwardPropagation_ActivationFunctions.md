A neural network is not useful just because it has layers.
It becomes useful because input values are transformed step by step as they move through those layers.

That process is called **forward propagation**.

---

### ***2) What Is Forward Propagation?***

Forward propagation is the process of taking the input data, passing it through each layer of the network, and producing an output.

Very roughly:

```text
input → hidden layer 1 → hidden layer 2 → ... → output layer
```

At each layer, the network does two main things:

```text
1) compute weighted sum + bias
2) apply activation function
```

So each layer performs:

```text
z = W*x + b
a = f(z)
```

Then the activation `a` becomes the input to the next layer.

That is the forward flow.

So if we write it in stages:

```text
Layer 1: z1 = W1*x + b1      a1 = f(z1)
Layer 2: z2 = W2*a1 + b2     a2 = f(z2)
Layer 3: z3 = W3*a2 + b3     a3 = f(z3)
...
Output: final result
```

This is forward propagation:
**input values move forward, get transformed at each step, and finally become a prediction.**

---

### ***3) Why “Forward” Propagation?***

It is called **forward** because the information flows from:
- input
- to hidden layers
- to output

Later, when we learn backpropagation, gradient information will flow in the reverse direction during training.

So:
- **forward propagation** = compute predictions
- **backpropagation** = compute how to update parameters

Day 3 is only about the first part:
how the network computes an output.

---

### ***4) Pre-Activation vs Activation***

We touched this on Day 2, but Day 3 is where it starts to matter operationally.

For one neuron:

```text
z = w1*x1 + w2*x2 + ... + wn*xn + b
a = f(z)
```

#### Pre-Activation
`z` is the **pre-activation** value.

It is the raw weighted sum plus bias, before applying the activation function.

#### Activation
`a` is the **activation**.

It is the transformed output after applying the activation function.

This distinction matters because:
- the network first builds a linear combination
- then applies a non-linear transformation
- that transformed result is what moves to the next layer

So:
- `z` = raw internal score
- `a` = activated output sent forward

This is the repeated heartbeat of the network.

---

### ***5) A Tiny Forward Pass Example***

Suppose one neuron has:
- inputs: `x1 = 2`, `x2 = 3`
- weights: `w1 = 0.5`, `w2 = -1`
- bias: `b = 2`

Then:

```text
z = 0.5*2 + (-1)*3 + 2
z = 1 - 3 + 2
z = 0
```

Now suppose the activation is ReLU:

```text
a = max(0, z) = max(0, 0) = 0
```

So this neuron outputs `0`.

That output would then be passed to the next layer, or used directly if this were the output layer.

That is a full forward step for one neuron.

---

### ***6) Activation Functions: Why We Need Them***

This is one of the most important ideas of Day 3.

Without activation functions, each layer would only do:

```text
z = W*x + b
```

That is linear.

And as we learned on Day 1, stacking linear transformations without non-linearity still collapses into one overall linear transformation.

So without activation functions:
- the network may look deep
- but mathematically it behaves like one linear model

That is why activation functions matter.

They introduce **non-linearity**.

That non-linearity lets the network:
- learn curved decision boundaries
- capture feature interactions
- build richer representations
- approximate more complex functions

So activation functions are not decorative details.
They are the reason depth becomes useful.

---

### ***7) ReLU***

ReLU stands for **Rectified Linear Unit**.

Its formula is:

```text
ReLU(z) = max(0, z)
```

This means:
- if `z > 0`, output `z`
- if `z <= 0`, output `0`

#### Example

```text
ReLU(5) = 5
ReLU(-3) = 0
ReLU(0) = 0
```

#### Why ReLU is popular
- simple to compute
- efficient in practice
- commonly used in hidden layers
- works well in many deep learning settings

#### Intuition
ReLU acts like a gate:
- negative values are shut off
- positive values pass through unchanged

#### Limitation
If a neuron keeps producing negative values, ReLU may keep outputting `0`, and that neuron can become inactive.
This is often called the **dying ReLU** issue.

For now, the main thing to remember is:
**ReLU is the standard default hidden-layer activation in many networks.**

---

### ***8) Sigmoid***

The sigmoid function is:

```text
sigmoid(z) = 1 / (1 + e^(-z))
```

Its output is always between `0` and `1`.

#### Example intuition
- very negative `z` → output near `0`
- `z = 0` → output `0.5`
- very positive `z` → output near `1`

#### Why it matters
Sigmoid is often used when we want an output that behaves like a probability, especially in **binary classification output layers**.

#### Limitation
For very large positive or negative values, sigmoid becomes very flat.
That can make learning slower because gradients become very small.

So sigmoid is important, but usually:
- less common in hidden layers of modern deep networks
- still very useful in binary classification outputs

---

### ***9) Tanh***

The tanh function is:

```text
tanh(z) = (e^z - e^(-z)) / (e^z + e^(-z))
```

Its output is between `-1` and `1`.

#### Example intuition
- very negative `z` → near `-1`
- `z = 0` → `0`
- very positive `z` → near `1`

#### Why tanh is useful
Compared with sigmoid:
- it is centered around `0`
- that can sometimes make optimization behavior nicer

#### Limitation
Like sigmoid, tanh can also saturate for large positive or negative values.

So tanh is important conceptually, even if ReLU is more common in many hidden-layer settings today.

---

### ***10) ReLU vs Sigmoid vs Tanh***

Here is the practical picture.

#### ReLU
- common in hidden layers
- simple
- fast
- works well in practice

#### Sigmoid
- output between `0` and `1`
- useful for binary classification output layers
- can saturate

#### Tanh
- output between `-1` and `1`
- zero-centered
- can also saturate

A useful memory trick:

- **ReLU**: keep positives, kill negatives
- **sigmoid**: squeeze into `0 to 1`
- **tanh**: squeeze into `-1 to 1`

### ***11) Logits vs Probabilities vs Predictions***

This distinction becomes important very quickly.

Suppose the final output layer gives some raw output value before any probability conversion.

That raw value is often called a **logit**.

#### Logit
A logit is a raw score produced by the model before turning it into a probability.

For binary classification, we often do:

```text
logit → sigmoid → probability
```

Example:

```text
logit = 2.0
probability = sigmoid(2.0) ≈ 0.88
```

#### Probability
A probability is the transformed value, usually between `0` and `1`, that expresses model confidence.

#### Prediction
A final prediction is the decision we make from the probability.

Example:
- probability = `0.88`
- threshold = `0.5`
- final class prediction = `1`

So:

```text
logit = raw score
probability = transformed confidence
prediction = final decision
```

These are related, but not the same thing.

---

### ***12) A Small Layer-Level Example***

Suppose:

```text
x = [1, 2]
```

And a hidden layer has two neurons.

Neuron 1:

```text
z1 = 1*1 + 2*2 + 0 = 5
a1 = ReLU(5) = 5
```

Neuron 2:

```text
z2 = (-1)*1 + 1*2 + 1 = 2
a2 = ReLU(2) = 2
```

So the hidden layer output becomes:

```text
a = [5, 2]
```

That vector is then passed to the next layer.

This is forward propagation at the layer level:
- input vector goes in
- each neuron computes its own `z`
- each activation is applied
- the layer outputs a new vector

### ***Simple Summary***

Forward propagation is the step-by-step process by which inputs move through a neural network and become outputs.

At each layer:
- compute a weighted sum plus bias
- apply an activation function
- pass the result to the next layer

Activation functions matter because they introduce non-linearity.
Without them, even a deep network behaves like one linear transformation.

ReLU is common in hidden layers.
Sigmoid is often used to turn binary output scores into probabilities.
Tanh outputs values between `-1` and `1`.

And for classification:
- **logit** = raw model score
- **probability** = transformed score
- **prediction** = thresholded final class
