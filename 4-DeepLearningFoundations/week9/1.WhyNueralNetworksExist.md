#### *1) **Where We Are Coming From***
</div>

Up to now, we have worked mostly with classical machine learning models.
Even when the algorithms differed, the overall structure was familiar:

**features go in → the model computes something structured → predictions come out → we evaluate the result.**

A plain linear model is one of the simplest examples of this idea.

For regression, we can write it as:

```text
y_hat = w1*x1 + w2*x2 + ... + wn*xn + b
```

This means:
- each feature gets a weight
- the model computes a weighted sum
- then it adds a bias term
- that becomes the prediction

That is elegant, useful, and often surprisingly strong.
Linear models are the sturdy wrench in the toolbox.

But they have a hard limit: they can only learn relationships that are **linear in the input space**.

---
<div style="color: gray;">

#### *2) **What “Linear” Really Means***
</div>

A model is linear when its output is built from a weighted combination of inputs without bending, folding, or twisting the space in a richer way.

- In two dimensions, a linear decision boundary is a **straight line**.
- In three dimensions, it is a **plane**.
- In higher dimensions, it is a **hyperplane**.

So if the real pattern in the data can be separated by a line or plane, a linear model can do well.

But many real patterns are not like that.

Examples:
- XOR logic cannot be solved by one straight line
- image patterns are not usually captured by one weighted sum of pixels
- language meaning is not a neat linear rule over word IDs
- complex tabular interactions often require combinations of effects, thresholds, and non-linear behavior

That is where the trouble begins.

---

<div style="color: gray;">

#### *3) **Why Linear Models Hit Limits***
</div>

Let’s make the problem concrete.

Suppose we want to classify points based on a pattern like this:
- **Class A:** points in the top-left and bottom-right
- **Class B:** points in the top-right and bottom-left

This is the classic XOR-type situation.

No single straight line can separate the classes cleanly.

A linear model tries to draw one dividing boundary.
But the structure of the data demands something more flexible.

This is the core motivation for neural networks:

> We need models that can learn more expressive functions.

Not random complexity. Not ceremonial suffering. Expressive functions.

---

<div style="color: gray;">

#### *4) **The Big Idea: Learned Representations***

</div>

A neural network does not usually solve a complex problem in one jump.
Instead, it transforms the input step by step.

```text
input → intermediate representation → better representation → output
```

This is the key leap.

Instead of asking the model to map raw input directly to the final answer in one shot, we let it build intermediate features internally.

That is why people say neural networks do **representation learning**.

This means the network learns useful internal transformations of the data instead of relying only on hand-crafted features.

Examples:
- in vision, early layers may learn edges or textures
- later layers may learn shapes or object parts
- in text, early transformations may capture token patterns
- later ones may capture semantic relationships
- in tabular data, hidden layers may learn combinations and interactions that were not explicit in the original features

So a neural network is not just “a bigger formula.”
It is a system that learns useful internal representations.

---
<div style="color: gray;">

#### *5) **First Mental Model of a Neural Network***
</div>

At a very high level, a neural network is a stack of layers.

Each layer:
- takes in numbers
- applies a learned transformation
- passes the result forward

A simple picture looks like this:

```text
x → Layer 1 → Layer 2 → Layer 3 → y_hat
```

Each layer has parameters:
- weights
- biases

Each layer computes something like:

```text
z = W*x + b
```

Then it usually applies a non-linear activation function:

```text
a = f(z)
```

That output becomes the input to the next layer.

So the pattern is:

```text
linear step → non-linear step → repeat
```

That is the engine.

---
<div style="color: gray;">

#### *6) **The Critical Truth: Stacking Linear Layers Alone Does Not Help***
</div>

This is one of the most important ideas in all of deep learning foundations.

Suppose we stack two linear transformations:

```text
h = W1*x + b1
y = W2*h + b2
```

Substitute the first into the second:

```text
y = W2(W1*x + b1) + b2
```

Expand it:

```text
y = (W2*W1)x + (W2*b1 + b2)
```

That is still just another linear transformation.

So two linear layers without activation are mathematically equivalent to one linear layer.

That means depth alone is not enough.

**This is the whole trick:** depth becomes useful only when we insert non-linearity between layers.

Without that, we are just wearing extra algebraic costume jewelry.

---
<div style="color: gray;">

#### *7) **Why Non-Linearity Changes Everything***
</div>

A non-linear activation function bends the mapping.

It lets the network create more complex boundaries and richer transformations.

With non-linearity, the model is no longer trapped inside one equivalent linear map.

This lets the network:
- model curved decision boundaries
- capture interactions between features
- build hierarchical internal representations
- approximate highly complex functions

This is why activation functions are not a side detail. They are foundational.

The rough story is:
- the linear part learns weighted combinations
- the non-linear part breaks the “just one linear map” limitation
- repeating this lets the network learn increasingly rich transformations

That is the conceptual birth of neural networks.

---
<div style="color: gray;">

#### *8) **Neural Networks Are Not Magic***
</div>

We should lock in the right mental model early, before the hype goblin starts tap dancing.

A neural network is:
- a parameterized function
- with many weights and biases
- organized in layers
- trained by optimization
- so that its predictions reduce some loss function

That’s it.

Very powerful, yes.
Mysterious, no.
Complex, often.
Magic, absolutely not.

The miracle is not supernatural.
The miracle is that gradient-based optimization over many parameters actually works as well as it does.

---
<div style="color: gray;">

#### *9) **Connecting This to What We Already Know***
</div>

Neural networks are not a complete break from earlier ML. They extend what we already know.

We already understand:
- features
- weighted sums
- optimization
- loss functions
- training vs validation behavior
- underfitting vs overfitting

Neural networks keep all of that.

What changes is:
- we now have many layers instead of one direct mapping
- we rely on learned internal representations
- we use non-linear activations
- we train many more parameters
- the training loop becomes more explicit

So deep learning is not “different physics.”
It is still machine learning.
Just with more expressive function classes and a more elaborate learning pipeline.

#### *10) **A Tiny Intuition Example***
</div>

Imagine we want to predict whether a house is “premium” or “not premium.”

A linear model might do something like:

```text
score = w1*(size) + w2*(location_score) + w3*(age) + b
```

That can work, but it assumes one direct weighted recipe.

A neural network can first learn hidden combinations like:
- “large and recently renovated”
- “small but in a top neighborhood”
- “older but luxury location”
- “unusual interaction between size and school district”

Then later layers combine those learned patterns into a final prediction.

This is not hand-coded.
The network learns these intermediate patterns from data.

That is the essence of learned representation.

#### *11) **What Day 1 Is Really Trying to Install in Our Heads***
</div>

If Day 1 works, we should leave with these five anchors:

1. Linear models are useful but limited.
2. Many real-world patterns require more expressive mappings.
3. Neural networks solve this by stacking learned transformations.
4. Stacking only linear transformations does not help.
5. Non-linearity is the ingredient that gives neural networks real power.

If those five ideas are solid, the rest of the week becomes much easier.

#### ***Simple Summary***
</div>

A linear model computes one weighted sum and makes a prediction from it.
That works when the pattern is simple and approximately linear.
But many problems are more complicated.

A neural network solves this by learning a sequence of transformations.
Each layer builds a new representation of the input.
But this only becomes powerful when non-linear activation functions are inserted between layers.
Otherwise, many layers still collapse into one linear transformation.

So the real story of neural networks is:

```text
weighted sums + non-linearity + layered representations + optimization
```

That is the skeleton.
